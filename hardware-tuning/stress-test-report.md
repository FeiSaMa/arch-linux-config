# Stress Test Report — 2026-06-12

## 测试条件

| 项 | 值 |
|------|--------|
| CPU | Intel Core Ultra 7 356H (16 核) |
| 模式 | performance (PL1=75W, PL2=110W) |
| 工具 | stress-ng --cpu 16 --timeout 600s |
| 时长 | 10 分钟 (600s) |
| 采样 | 每 10s 一次，共 60 次 |

## 温度

| 统计 | 值 |
|------|-----|
| 初始 | 58°C |
| 峰值 | **97°C** |
| 稳定 | 96~97°C |
| 平均 | 95.85°C |

TJmax=100°C，持续负载下留有约 3~4°C 余量。

## CPU 频率

| 统计 | 值 |
|------|-----|
| 初始爆发 | **4.01 GHz**（PL2 窗口） |
| 稳定平均 | **3.00 GHz** |
| 最低稳定 | ~2.59 GHz |

CPU 频率曲线：
- 前 10s：4.0 GHz（短暂爆发）
- 10~20s：降至 3.5 GHz（温度达 95°C）
- 20s 后：稳定在 2.9~3.1 GHz（PL1 功率墙限制）

## GPU 频率

全程稳定在 700 MHz（min_freq 地板），无 throttling。

## 降频结论

**存在降频，但属正常 PL1 功率墙行为，非异常 thermal throttling：**

1. **PL2 爆发阶段** — 前 10s 达 4.0 GHz，然后受 75W PL1 限制回落
2. **PL1 稳态** — 75W 功率墙下 16 核稳定约 3.0 GHz，每核约 4.7W
3. **温度边界** — 96~97°C 持续运行，未触 TJmax (100°C)，无强制降频
4. **throttle flags** — 全程 GPU throttle=0，未触发任何硬件节流

**结论：当前 75W PL1 设置是合理的热边界。** 如果希望提升持续多核性能，需要进一步提高 PL1（但已接近 TJmax，收益有限且风险增加）。对单核/轻多核场景，CPU 仍可自由 boost 到 4.7 GHz。
